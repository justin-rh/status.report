## Requirements — v2.0

### App Detection

- [x] **APP-V2-01**: User can see whether Company Portal (UWP) is installed on a Windows device, with MDM enrollment status shown in the Service column of that row
  - *Outcome:* Implemented via `_detect_mdm_enrollment()` + `HKLM\Enrollments` UPN check. Live machine validation deferred (hardware-gated).

### Warnings

- [x] **WARN-01**: HTML sheet warns when the device runs Windows 10 or earlier (OS build < 22000)
  - *Outcome:* Implemented in `health_checks.evaluate_warnings()`; 17 parametrized tests pass.
- [x] **WARN-02**: HTML sheet warns when disk free space is ≤ 15% of total capacity
  - *Outcome:* Implemented in `health_checks.evaluate_warnings()` with boundary tests.
- [x] **WARN-03**: Warnings appear in a collapsible box at the bottom of the sheet; each check shows OK or WARN status; box is collapsed with a green "All checks passed" header when all pass, auto-expanded when any warning fires
  - *Outcome:* `<details open>` auto-expand with `has_warnings` flag; renderer + template tests confirmed. ✓

### Mac Platform

- [x] **PLAT-V2-01**: Tool collects Mac hardware stats — CPU model (Intel and Apple Silicon), total RAM, disk capacity and free space, macOS version
  - *Outcome:* `collectors/mac/hardware.py` with `sysctl`/`system_profiler` dispatch. Live Mac validation deferred.
- [x] **PLAT-V2-02**: Tool enumerates all local user profiles on macOS (non-system accounts, UID ≥ 501)
  - *Outcome:* `collect_profiles()` via `pwd` module with UID filter. Live Mac validation deferred.
- [x] **PLAT-V2-03**: Tool detects the following apps on macOS: NinjaOne, CrowdStrike Falcon, Microsoft 365, Zoom, Google Chrome, Claude Desktop, Company Portal
  - *Outcome:* `collectors/mac/apps.py` with `MAC_APP_SPECS`; plistlib-based detection. NinjaOne launchctl label at LOW confidence. ✓ (automated tests pass)
- [x] **PLAT-V2-04**: Same D&D HTML character sheet is rendered and saved on macOS runs (output to `logs/` relative to exe)
  - *Outcome:* Platform dispatch in `collect_all()` and `main.py`; Mac output path via `Path(sys.executable).parent`. Live Mac validation deferred.

### NinjaOne Compatibility

- [x] **NINJA-01**: Exe runs without crashing or hanging under SYSTEM account (no interactive prompts, no display or browser dependency)
  - *Outcome:* `sys.stdin.isatty()` guard on `os.startfile()` and `input()`; tested headless in CI. Live SYSTEM-account run deferred.
- [x] **NINJA-02**: Key audit stats printed to stdout after each run (hostname, OS version, CPU, RAM, disk %, active warning count) for NinjaOne script log capture
  - *Outcome:* `[SUMMARY]` print outside `isatty()` guard; emitted on every run. Live NinjaOne log capture validation deferred.
