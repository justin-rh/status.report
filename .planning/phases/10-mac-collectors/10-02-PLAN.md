**_detect_bundle(app_bundle: str) -> tuple[bool, str | None]**:
- `bundle_path = APPLICATIONS_DIR / app_bundle`
- `if not bundle_path.exists(): return False, None`
- `plist_path = bundle_path / "Contents" / "Info.plist"`
- `with plist_path.open("rb") as f: data = plistlib.load(f)` — BINARY MODE "rb" (Pitfall 2)
- `return True, data.get("CFBundleShortVersionString")`
- Catch `(OSError, plistlib.InvalidFileException, KeyError, Exception)` → `return True, None` (bundle exists but version unreadable)

**_query_launchd(label: str) -> str**:
- `subprocess.run(["launchctl", "list", label], capture_output=True, text=True, timeout=5)`
- `return "Running" if result.returncode == 0 else "Stopped"`
- Catch `(OSError, subprocess.TimeoutExpired)` → `return "Stopped"`
- NOTE: Exit code 0 means daemon found in launchd registry; does NOT guarantee active PID.
  Standard user cannot reliably query system daemons on macOS Monterey+ without sudo.
  Treat as best-effort (RESEARCH Pitfall 6, D-17).

**_detect_one_app(spec: dict, report: AuditReport) -> None**:
Detection precedence:
1. Directory check if `"app_dir" in spec` (NinjaOne): `(APPLICATIONS_DIR / spec["app_dir"]).is_dir()` → installed=True, version=None
2. Bundle check if `"app_bundle" in spec` (and not already installed): call `_detect_bundle(spec["app_bundle"])`
3. Fallback bundles if `"fallback_bundles" in spec` (and not yet installed): iterate and call `_detect_bundle()` on each, break on first hit
4. LaunchDaemon plist fallback if `"launchdaemon_plist" in spec` (and not yet installed): `(LAUNCH_DAEMONS_DIR / spec["launchdaemon_plist"]).exists()` → installed=True, version=None
5. Service state if `installed` and `"launchdaemon_label" in spec`: call `_query_launchd(spec["launchdaemon_label"])`
6. Set `detection_method = "filesystem"` for all (Mac detection is always filesystem/launchd based)
7. `report.apps.append(AppStatus(name=spec["name"], installed=installed, version=version, service_state=service_state, detection_method=detection_method))`

**detect_apps(report: AuditReport) -> None** (always-append rule — D-16):

for spec in MAC_APP_SPECS:

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /Applications/ filesystem → app detection | Bundle existence and plist content trusted (local system state) |
| launchctl stdout/exit code → service_state | Trusted local system call; not used for security decisions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Denial of Service | subprocess launchctl calls | mitigate | timeout=5 on all subprocess.run calls; caught in except clause |
| T-10-02-02 | Spoofing | plistlib.load() reading Info.plist | accept | An attacker who can write /Applications/ already has higher access than this tool; tool is audit-only |
| T-10-02-03 | Denial of Service | plistlib.load() on malformed plist | mitigate | Catch (OSError, plistlib.InvalidFileException, KeyError, Exception) → return (True, None); detection continues |
</threat_model>
