**Step 1: Copy the distributable to a USB flash drive**
Copy the entire `dist\status_report\` folder to a USB flash drive. The USB should contain:
```
status_report\
    status_report.exe
    _internal\
        ... (supporting files)
```

**Step 2: Run from USB on a CrowdStrike Falcon-enrolled machine**
On a Master Electronics Windows 10 or 11 machine enrolled in CrowdStrike Falcon:
1. Log in as a STANDARD USER (not admin — PKG-01 requirement)
2. Open the USB drive in File Explorer
3. Double-click `status_report.exe`
4. Watch for: CrowdStrike quarantine alert, Windows SmartScreen prompt, or other security block

**Step 3: Verify the four success criteria (ROADMAP Phase 5)**
If exe runs without being blocked:
- SC1: Console shows progress and exits within 30 seconds
- SC2: HTML file appears in `USB:\status_report\logs\status_{HOSTNAME}_{DATE}.html`
- SC3: Open the HTML file in a browser — verify the character sheet renders correctly with the machine's hostname, hardware stats, and app detection results
- SC4: Confirm no files were written to C:\, %TEMP%, or %APPDATA% (check Task Manager → process gone; check %TEMP% for any new files)

**Step 4: Check bundle size on USB**
Right-click `status_report\` folder on USB → Properties → confirm total size < 50 MB

**Step 5: Record the result**
Update `.planning/ROADMAP.md` Phase 5 Success Criterion 4 to record the test result:
- PASS: "CrowdStrike Falcon test passed on [MACHINE] ([DATE]) — no quarantine, no block. Distribution approved."
- QUARANTINE: Follow D-12 — request path/hash exclusion from CrowdStrike tenant admin, OR escalate to evaluate code-signing (DIST-V2-01). Return with "quarantined — escalating per D-12" and describe the specific block behavior.

**If the exe is quarantined:**
Do NOT type "approved". Type "quarantined" and describe what CrowdStrike showed (alert text, prevention action: detect vs block vs quarantine). Claude will help document the D-12 escalation path.

**If the exe ran but HTML is in the wrong location (e.g., C:\ instead of USB):**
Do NOT type "approved". Type "wrong output path: [actual path]". This indicates a sys.executable issue — Claude will diagnose.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| USB exe → CrowdStrike-enrolled host | CrowdStrike behavioral engine inspects the exe at launch — --onedir and upx=False reduce false positive risk |
| USB logs/ → IT staff | HTML files accumulate on USB; physical USB security is outside code scope |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Tampering | dist/status_report/ on USB | accept | USB is physically carried by IT; no code-level mitigation. Code signing (DIST-V2-01) is the v2 answer. |
| T-05-03-02 | Denial of Service | CrowdStrike quarantine blocks exe | mitigate | D-11 test run before distribution; D-12 escalation path (exclusion by hash/path) if blocked. build uses --onedir + upx=False to minimize behavioral detection surface. |
| T-05-03-03 | Repudiation | CrowdStrike test not recorded | mitigate | D-13 requires ROADMAP.md SC4 be updated with test result before distribution. This task enforces that gate. |
</threat_model>
