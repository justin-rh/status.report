## Color

60/30/10 split is established by the existing design. Phase 13 adds one new
semantic class within the existing palette.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a2e` (`--bg-dominant`) | Page background |
| Secondary (30%) | `#16213e` (`--bg-secondary`) | Section cards, alternating table rows |
| Accent (10%) | `#0f3460` (`--bg-accent`) | Section title bars, table header rows |
| OK / Installed | `#22c55e` (`--green`) | `.badge-installed` — app installed, health OK |
| Caution / Yellow | `#f59e0b` (`--amber`) | `.badge-warn` — uptime 7–30 days (`level='yellow'`) |
| Critical / Red | `#ef4444` (`--red`) | `.badge-missing`, new `.badge-critical` — uptime > 30 days (`level='red'`) |
| Muted | `#6b7280` (`--text-muted`) | "N/A" values, unavailable data |

**Accent reserved for:** Section title bars and table column header backgrounds
only. Never used for interactive controls, badges, or stat values.

**New color rule for Phase 13:**
- Warning badge class is determined by `w.level` when `w.severity == 'WARN'`:
  - `w.level == 'yellow'` → use `.badge-warn` (`--amber`)
  - `w.level == 'red'` → use `.badge-critical` (`--red`, new class)
  - `w.level is None` and `w.severity == 'OK'` → use `.badge-installed` (`--green`)

**New CSS class to add (`.badge-critical`):**
```css
.badge-critical {
  background: var(--red);
  color: #fff;
}
```
This is the only new CSS class in Phase 13.

| Warnings box heading — no warnings | "&#10003; Health Checks" | Existing template (unchanged) |

**Primary CTA:** No CTA in this phase. The output is a read-only HTML report.

**Empty state:** "N/A" displayed in `.stat-value.muted` style when data is
unavailable (standard user cannot access WUA COM). The `—` em-dash is used when
a value was never collected (uptime collection failure only).

**Error state:** Collection errors are appended to `report.collection_errors`
(existing pattern) and do not surface in the HTML UI. The stat row shows "—"
or "N/A" silently. No error message is displayed to the user in the HTML output.

**Destructive actions:** None. The character sheet is a read-only output file.
No confirmations required.

## Component Inventory for Phase 13

These are the specific HTML/CSS constructs Phase 13 must add or modify.

### New: System Health Stat Block Section

Insert after the "Other Profiles" row and before `</div>` closing the `.stat-grid`.
This section header uses a visual divider pattern to group the two new rows.

```html
<!-- System Health rows — Phase 13 -->
<div class="stat-label">Uptime</div>
<div class="stat-value{% if uptime_display is none %} muted{% endif %}">
  {{ uptime_display | default('—', true) }}
</div>

<div class="stat-label">Pending Updates</div>
<div class="stat-value{% if pending_updates_display == 'N/A' %} muted{% endif %}">
  {{ pending_updates_display }}
</div>
```

**Visual contract:**
- Label text: 12px/600/`--text-heading`, uppercase, 0.05em letter-spacing
- Value text: 14px/400/`--text-body`; muted (`--text-muted`) when value is "—" or "N/A"
- Row padding: `8px 0` (matching all other stat rows)
- Grid column widths: 160px label / 1fr value (unchanged stat-grid layout)

### Modified: Warning Badge in Warnings Box

Update the badge `<span>` in the warnings box `{% for w in warnings %}` loop to
apply `badge-critical` for `level='red'` warnings.

**Before (existing):**
```html
<span class="badge {% if w.severity == 'WARN' %}badge-warn{% else %}badge-installed{% endif %}">
  {% if w.severity == 'WARN' %}&#9888; WARN{% else %}&#10003; OK{% endif %}
</span>
```

**After (Phase 13):**
```html
<span class="badge {% if w.severity == 'WARN' %}{% if w.level == 'red' %}badge-critical{% else %}badge-warn{% endif %}{% else %}badge-installed{% endif %}">
  {% if w.severity == 'WARN' %}&#9888; WARN{% else %}&#10003; OK{% endif %}
</span>
```

**Visual contract:**
- `level='yellow'` warnings: amber badge (`--amber` / `#f59e0b`) — unchanged
- `level='red'` warnings: red badge (`--red` / `#ef4444`) — new `.badge-critical`
- `level=None` / `severity='OK'`: green badge (`--green` / `#22c55e`) — unchanged
- Badge text always reads "WARN" or "OK" — level does not change the badge text

### Auto-Expand Behavior (unchanged)

`has_warnings = any(w.severity == 'WARN' for w in report.warnings)` in
`renderer/__init__.py` line 167. Both UPTIME_WARN (`level='yellow'`) and
UPTIME_STALE (`level='red'`) have `severity='WARN'`, so the warnings box
auto-expands for both thresholds. No change needed to this computation.

## State Map

| Data state | Uptime display | Updates display | Warning badge |
|------------|---------------|-----------------|---------------|
| Uptime collected, ≤ 7 days | "N days H hours" | per updates state | green OK |
| Uptime collected, 7–30 days | "N days H hours" (muted: no) | per updates state | amber WARN |
| Uptime collected, > 30 days | "N days H hours" (muted: no) | per updates state | red WARN (badge-critical) |
| Uptime collection failed | "—" (muted) | per updates state | green OK |
| Updates collected, SYSTEM/Admin | "N pending" | — | (no badge — informational) |
| Updates unavailable, standard user | "N/A" (muted) | — | (no badge — informational) |
