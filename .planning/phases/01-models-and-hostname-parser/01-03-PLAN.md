Disambiguation algorithm (RESEARCH.md Architecture Patterns, lines 146-172):
  Step 1: Split on '-', uppercase all parts: parts = hostname.upper().split('-')
          Store original input in raw_hostname BEFORE uppercasing (D-05)
  Step 2: parts[0] not in CITY_CODES -> return ParsedHostname(raw_hostname=hostname, device_type='Unknown')  [D-04]
  Step 3: len(parts) < 3 -> return ParsedHostname(raw_hostname=hostname, city=city, device_type='Unknown')   [D-09]
  Step 4: seg2 = parts[1], seg3 = parts[2]
  Step 5: seg2 in P3_CODES (P3A/P3B/P3C) -> P3 Warehouse Device                     [checked FIRST — Pitfall 1]
  Step 6: 'LAP' in seg2 -> Department Laptop, department=seg2
  Step 7: seg3.isdigit() -> Warehouse Workstation, department=seg2, station=int(seg3) [D-01, D-02, D-06]
  Step 8: seg3.isalpha() and seg3.isupper() -> User-Assigned Laptop, company_code=seg3 [D-01, D-03]
  Step 9: fallback -> ParsedHostname(raw_hostname=hostname, city=city, device_type='Unknown')

Critical pitfalls (from RESEARCH.md Common Pitfalls):
  Pitfall 1: If seg3.isdigit() check runs before P3 check, PHX-P3A-001 -> Warehouse Workstation (WRONG)
             FIX: P3_CODES check is ALWAYS first
  Pitfall 2: station='003' instead of station=3 — use int(seg3), not seg3
  Pitfall 3: lowercase input fails lookup — uppercase before lookup, preserve original for raw_hostname
  Pitfall 5: Missing parsers/__init__.py causes ModuleNotFoundError

      def parse_hostname(hostname: str) -> ParsedHostname:
          """Pure function: hostname string -> ParsedHostname. Never raises.

          Decision rules D-01 through D-09 from CONTEXT.md apply.
          Anti-pattern: do NOT check seg3.isdigit() before seg2 in P3_CODES.
          """
          parts = hostname.upper().split('-')

          # D-04: unrecognized city prefix -> Unknown, silent, no further parsing
          if not parts or parts[0] not in CITY_CODES:
              return ParsedHostname(raw_hostname=hostname, device_type='Unknown')

          city = CITY_CODES[parts[0]]

          # D-09: recognized city but not enough segments for type determination
          if len(parts) < 3:
              return ParsedHostname(raw_hostname=hostname, city=city, device_type='Unknown')

          seg2, seg3 = parts[1], parts[2]

          # P3 Warehouse Device — MUST be checked BEFORE seg3.isdigit() (Pitfall 1)
          if seg2 in P3_CODES:
              return ParsedHostname(
                  raw_hostname=hostname, city=city,
                  device_type='P3 Warehouse Device',
                  station=_parse_station(seg3),
              )

          # Department Laptop — LAP substring in seg2 (D-01)
          if 'LAP' in seg2:
              return ParsedHostname(
                  raw_hostname=hostname, city=city,
                  device_type='Department Laptop',
                  department=seg2,
                  station=_parse_station(seg3),
              )

          # Warehouse Workstation — seg3 is all digits (D-01, D-02, D-06)
          # D-06: no whitelist on dept code — any seg2 qualifies
          if seg3.isdigit():
              return ParsedHostname(
                  raw_hostname=hostname, city=city,
                  device_type='Warehouse Workstation',
                  department=seg2,
                  station=int(seg3),
              )

          # User-Assigned Laptop — seg3 is alphabetic uppercase (D-01, D-03)
          # D-03: unrecognized company code still yields User-Assigned Laptop
          if seg3.isalpha() and seg3.isupper():
              return ParsedHostname(
                  raw_hostname=hostname, city=city,
                  device_type='User-Assigned Laptop',
                  company_code=seg3,
              )

          # Ambiguous structure with recognized city (D-09 extension)
          return ParsedHostname(raw_hostname=hostname, city=city, device_type='Unknown')

      def _parse_station(seg: str) -> int | None:
          """Convert segment to int station number; return None if not numeric."""
          try:
              return int(seg)
          except (ValueError, TypeError):
              return None

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External hostname string → parser | The raw hostname from socket.gethostname() (Phase 2) enters here. It is untrusted user-visible data on an IT tool — hostile hostnames must never execute. |
| Parser output → renderer | ParsedHostname fields flow into the Jinja2 HTML template in Phase 3. HTML escaping is the renderer's responsibility, not the parser's. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Tampering | parse_hostname() — hostname injection | mitigate | parse_hostname() is a pure function: str.split() + dict lookup + isdigit()/isalpha() comparisons only. No subprocess, no eval, no exec, no shell expansion. A hostname containing shell metacharacters (semicolons, backticks, pipes) is split into segments and classified as Unknown — it never reaches an OS call. |
| T-01-03-02 | Denial of Service | parse_hostname() — pathological input | accept | Input comes from socket.gethostname() on Windows — max hostname length is 15 characters (NetBIOS) to 253 characters (DNS). No unbounded regex backtracking; split('-') is O(n). Risk is negligible. |
| T-01-03-03 | Information Disclosure | CITY_CODES dict — office locations | accept | City code mappings are visible in source. This is an internal IT tool run by IT staff; the city list is not sensitive relative to the tool's threat model. |
| T-01-03-04 | Spoofing | Unverified hostname | accept | The tool accepts the hostname from the OS at face value. A user who renames their PC can spoof a different city/device type. Out of scope — this is an audit-assist tool, not an enforcement mechanism. Document in user-facing notes if needed. |
</threat_model>
