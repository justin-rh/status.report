### Non-critical observations

1. `writers.write_html` (writers/__init__.py:16) is dead code from the user pipeline — `main.py` calls `render_html()` directly. The hard-coded `scry.html` filename in the dead path cannot affect output, but the unused path is misleading. **Tech debt for v3.1.**

2. `_run_cli` mode (`--name`/`--serial`/`--warnings`) honors `--updates` and runs `collect_pending_updates` + `collect_vendor_updates`, but the collected data is never used (the early-exit prints only hostname/serial/warning messages). **Wasted work, not a bug.**

3. `--app NAME --output PATH` silently discards `--output` (matches D-08/D-13: `--app` never writes files). **By design; spec acknowledges this.**
