## Milestone Goal

Surface system health signals and extend CLI output options so IT staff can assess machine state and integrate output into NinjaOne workflows.

## Requirements (final state)

### Rename

- [x] **RENAME-01**: All source files, build spec, `build.bat`, and planning docs reference SCRY instead of StatusReport; `scry.exe` and `scry.spec` replace `status_report.exe` and `status_report.spec` — *delivered Phase 12*
- [x] **RENAME-02**: Output filename format changed from `status_{hostname}_{date}.html` to `{date}_scry_{hostname}.html` (date-first for alphabetical sort, `scry` replaces `status`) — *delivered Phase 12*

### System Health

- [x] **HEALTH-01**: User can see pending Windows update count in the character sheet stat block; degrades gracefully to N/A when running as standard user (WUA COM requires SYSTEM or Administrator) — *delivered Phase 13 (live SYSTEM/Admin run deferred to hardware UAT)*
- [x] **HEALTH-02**: User can see uptime since last reboot in the character sheet stat block (days + hours display format) — *delivered Phase 13 (live machine run deferred to hardware UAT)*
- [x] **WARN-04**: Tool emits a yellow (caution) warning when uptime > 7 days; `UPTIME_WARN_DAYS = 7` configurable constant in `health_checks.py` — *delivered Phase 13 (live > 7d machine deferred to hardware UAT)*
- [x] **WARN-05**: Tool emits a red (critical) warning when uptime > 30 days; `UPTIME_STALE_DAYS = 30` configurable constant in `health_checks.py`; warning text notes that hibernation time is counted on Windows — *delivered Phase 13 (live > 30d machine + visual red badge deferred to hardware UAT)*

### Vendor Update Detection

- [x] **VENDOR-01**: User can see Dell Command Update installation status and pending update count in the character sheet; count read passively from `DCUApplicableUpdates.xml` (never invokes `dcu-cli.exe`); shows "Not installed" when DCU is absent — *delivered Phase 14 (live Dell machine + IT registry path confirmation deferred)*
- [x] **VENDOR-02**: User can see Lenovo System Update installation status in the character sheet; pending count shown as N/A (no passive source available in v3.0); never invokes `tvsu.exe` — *delivered Phase 14 (live Lenovo machine + IT registry path confirmation deferred)*

### Extended CLI Output

- [x] **OUT-V3-01**: `--json` flag serializes full `AuditReport` to a JSON file in `logs/` alongside the HTML report; uses `dataclasses.asdict()` + `json.dumps()` — *delivered Phase 15*
- [x] **OUT-V3-02**: `--output <path>` flag overrides the default `logs/` destination for HTML and JSON output; any writable path is accepted (D-02/D-03 — host-path validation removed per user decision) — *delivered Phase 15 (deviation: host-path validation removed mid-planning)*
- [x] **CLI-V3-01**: `--app <name>` runs only the app-detection pipeline for one named app and prints result to stdout; `--app + --json` produces a single-app JSON blob to stdout; app name matching is case-insensitive — *delivered Phase 15*

## Implementation Notes (final)

- **Warning severity:** `Warning.level: str | None` added as LAST field in dataclass to preserve backward-compat with positional `Warning(code, severity, message, detail)`. Both yellow and red levels still trigger `<details open>` auto-expand of the warnings box.
- **WUA privilege:** Works under SYSTEM (NinjaOne execution) and Administrator. Standard-user interactive runs degrade to `pending_updates = None` with a CollectionResult error. Live SYSTEM-context test deferred to hardware UAT.
- **DCU XML staleness:** Returns "Unknown (no scan data)" rather than 0 when `DCUApplicableUpdates.xml` is absent — distinguishes no-data from 0 pending.
- **`_WIN32COM_AVAILABLE` guard:** Mirror of `_WMI_AVAILABLE`/`_PWD_AVAILABLE` pattern — enables CI testing without a COM server. Named after the library (not the feature) for future reuse.
- **PyInstaller:** `--hidden-import win32timezone` added to `scry.spec` to bundle pywin32 timezone resources.
- **OUT-V3-02 deviation:** Original spec required validation that resolved path does not write to the host PC. Removed per D-02/D-03 — runs from USB but host-PC writes are now user-controlled, not blocked. ROADMAP.md SC2 reads "any writable path is accepted".

## Out of Scope (v3.0)

| Feature | Reason |
|---------|--------|
| LSU pending count via `tvsu.exe` | Requires admin elevation, has side effects, violates PKG-02 |
| DCU pending count via `dcu-cli.exe` | Requires admin elevation, may trigger downloads, violates PKG-02 |
| WUA update listing (titles / KB numbers) | Count is sufficient for the IT audit use case |
| `--json` as pipeline mode selector | JSON is an output-format modifier only; does not skip collection |
