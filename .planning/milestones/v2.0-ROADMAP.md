## Milestone Summary

**Key Decisions:**
- `sys.stdin.isatty()` as single headless guard — wraps `os.startfile()` and `input()` only
- `[SUMMARY]` line outside `isatty()` guard — always emitted regardless of interactive/headless mode
- `_detect_mdm_enrollment()` runs unconditionally — enrollment visible even under SYSTEM account with no HKCU
- `platform.machine()` dispatch (not `sys.platform`) for Intel vs Apple Silicon CPU collection
- `argparse` branch at top of `main()` with early return — no changes to existing full-pipeline path
- `patch("sys.argv")` in `_patched_main` helper — shared fix for argparse + pytest argv conflict

**Issues Resolved:**
- NinjaOne SYSTEM-account hang: `isatty()` guard on all interactive calls
- `HKCU` registry access under SYSTEM: already handled by `_detect_msix()` exception handling (SC4 no-code-change)
- argparse consuming pytest argv on full suite run: `sys.argv` patch in `_patched_main` helper
