## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AuditReport.warnings → Jinja2 template | Warning.message and Warning.detail strings cross into HTML rendering |
| main.py pipeline | evaluate_warnings called with report populated by all collectors |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Injection (XSS) | Warning.message / Warning.detail rendered in HTML template | mitigate | Jinja2 Environment has autoescape=True (already enforced in renderer/__init__.py Environment constructor); all Warning strings are auto-escaped on output; no raw HTML in Warning fields |
| T-07-02-02 | Tampering | has_warnings computed from report.warnings in _build_context | accept | has_warnings is derived from Warning.severity == 'WARN' — pure Python bool, no external input; only consequences are auto-expanding the collapsible box (cosmetic) |
| T-07-02-03 | Information Disclosure | Warning.detail contains hostname and OS build in HTML output | accept | hostname and OS build already visible in the header section; no new exposure |
| T-07-02-04 | Denial of Service | evaluate_warnings called before render in main.py | accept | evaluate_warnings is a pure function with no I/O; it cannot hang, crash, or consume unbounded resources; never-raise contract established in Phase 6 |
</threat_model>
