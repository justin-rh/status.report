<objective>
Fix two independent misbehaviors in main.py:

**DEBT-02:** `_run_cli()` (the `--name`/`--serial`/`--warnings` path) contains a dead collector block that calls `collect_pending_updates` and `collect_vendor_updates` when `--updates` is set. These results are never used in the CLI output path — `evaluate_warnings()` only checks OS version, disk space, rename, and uptime. Running SCRY with `--name --updates` wastes time collecting update data that is discarded immediately.

**DEBT-03:** When a user runs `scry --app chrome --output C:\temp`, SCRY silently ignores `--output` because app-query mode never writes files. The user gets no indication their flag was discarded. Adding a stderr warning makes the behavior explicit.

Purpose: Both fixes bring the CLI into alignment with what it actually does — no silent waste, no silent flag discard.
Output: main.py modified; no new files; all existing tests continue to pass.
</objective>
