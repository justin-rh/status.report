## Overview

StatusReport v1.0 is a self-contained Windows .exe that runs from a USB flash drive and performs an IT audit of the host PC. It decodes the device name using Master Electronics' naming convention (city + device type + department + station), collects system info and installed software status, and presents everything as a D&D-style HTML character sheet — while writing the output back to the flash drive. Packaged with PyInstaller --onedir and validated on a CrowdStrike Falcon-enrolled machine as a standard user.

## Milestone Summary

**Key Decisions:**

- PyInstaller `--onedir` chosen over `--onefile` — `--onefile` quarantined by CrowdStrike Falcon
- `Win32_Product` WMI class explicitly prohibited — triggers MSI reconfiguration side effects
- Output path from `Path(sys.executable).parent` — USB-only, no host writes
- M365 detected as single suite entry (D-05) rather than 5 individual app entries
- MERP detection: filesystem-first at hardcoded PVX Plus path; registry search for version on filesystem hit
- CrowdStrike display names: 'CrowdStrike Windows Sensor'/'CrowdStrike Sensor Platform' (verified from live registry)
- Claude MSIX via AppModel repository family prefix (primary) + Uninstall key sweep (fallback)
- `ir.files('renderer').joinpath('templates/character_sheet.html')` — single-string joinpath required for PyInstaller compatibility
- `autoescape=True` on Jinja2 Environment — no manual escaping required in templates

**Issues Resolved:**

- CrowdStrike Falcon `--onedir` quarantine risk — resolved: `--onedir` + `upx=False` sufficient, no exclusion or code signing needed
- MERP registry path unknown — resolved: filesystem-first detection at confirmed PVX Plus path
- WMI COM server unavailability in CI — resolved: `_wmi_module`/`_WMI_AVAILABLE` module-level import guard
