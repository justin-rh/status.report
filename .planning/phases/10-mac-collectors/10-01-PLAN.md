**File structure** (mirror collectors/windows/hardware.py):

```python
"""macOS hardware and profile collectors.
Implements PLAT-V2-01 (hardware stats) and PLAT-V2-02 (local user profiles).
Both functions mutate AuditReport in place and never raise (D-01, D-02 from 02-CONTEXT.md).
"""
from __future__ import annotations

import json
import os
import platform
import subprocess

import psutil

from models import AuditReport

# Module-level pwd import guard — mirrors _wmi_module pattern from windows/hardware.py.
# pwd is POSIX-only; this guard prevents ImportError on Windows CI (RESEARCH Pitfall 4).
try:
    import pwd as _pwd_module
    _PWD_AVAILABLE = True
except ImportError:
    _pwd_module = None  # type: ignore[assignment]
    _PWD_AVAILABLE = False
```

**Public interface** (never raises, in-place mutation):
```python
def collect_hardware(report: AuditReport) -> None:
    _collect_os(report)
    _collect_cpu_model(report)
    _collect_memory_and_disk(report)
    _collect_current_user(report)

def collect_profiles(report: AuditReport) -> None:
    try:
        report.local_profiles = _enumerate_profiles()
    except Exception as exc:
        report.collection_errors.append(f"Profile enumeration failed: {exc}")
```

**_collect_os** (D-06): Two separate subprocess.run calls.
- `subprocess.run(["sw_vers", "-productVersion"], capture_output=True, text=True, timeout=5)` → strip() → prefix with "macOS " → report.os_version
- `subprocess.run(["sw_vers", "-buildVersion"], capture_output=True, text=True, timeout=5)` → strip() → report.os_build
- Each in its own try/except Exception block appending `f"OS version/build collection failed: {exc}"` to collection_errors

**_collect_cpu_model** (D-07) — TWO-BRANCH detection:
- `arch = platform.machine()` — "arm64" on Apple Silicon, "x86_64" on Intel
- Intel path (arch == "x86_64"): `subprocess.run(["sysctl", "-n", "machdep.cpu.brand_string"], capture_output=True, text=True, timeout=5)` → strip() or None → report.cpu_model → return
- Apple Silicon path (arm64, or Intel sysctl fallback): `subprocess.run(["system_profiler", "SPHardwareDataType", "-json"], capture_output=True, text=True, timeout=10)` → json.loads → data["SPHardwareDataType"][0] → `hw.get("chip_type") or hw.get("cpu_type") or None` → report.cpu_model
- Each branch wrapped in try/except Exception appending `f"CPU model (sysctl) failed: {exc}"` or `f"CPU model (system_profiler) failed: {exc}"`
- NOTE: Do NOT call sysctl on arm64 — machdep.cpu.brand_string is Intel-specific (RESEARCH Pitfall 1)

**_collect_memory_and_disk** (D-08, D-09):
- `report.ram_gb = round(psutil.virtual_memory().total / (1024 ** 3), 1)` — no try/except (never fails at standard user)
- `disk = psutil.disk_usage("/")` in try/except → report.disk_total_gb and report.disk_free_gb (root partition, NOT "C:\\")
- Exception: `report.collection_errors.append(f"Disk usage collection failed: {exc}")`

**_collect_current_user** (D-10):
- `report.current_user = os.environ.get("USER") or os.environ.get("USERNAME") or None`
- Mac uses USER env var (not USERNAME). No try/except needed — os.environ.get() never raises.

**_enumerate_profiles** (D-11):
- Guard at top: `if not _PWD_AVAILABLE or _pwd_module is None: raise RuntimeError("pwd module not available on this platform")`
- `for entry in _pwd_module.getpwall():` — filter `entry.pw_uid >= 501` → `profiles.append(entry.pw_name)`
- Returns list[str]; raises on failure — caller (collect_profiles) catches it
- UID threshold is 501 (first human account on macOS; system accounts are < 501)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| subprocess → macOS CLI tools | sw_vers, sysctl, system_profiler output is trusted (local system calls, no user input) |
| pwd database → report | getpwall() reads local Open Directory; data is trusted system state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01-01 | Denial of Service | subprocess calls (sw_vers, sysctl, system_profiler) | mitigate | timeout=5/10 on all subprocess.run calls; TimeoutExpired caught in except Exception |
| T-10-01-02 | Information Disclosure | local_profiles (pw_name list) | accept | Same data already collected on Windows; tool is run by IT staff; profiles written to USB only |
| T-10-01-03 | Tampering | subprocess stdout content (system_profiler JSON) | accept | Attacker would need root to tamper with system_profiler output; tool is audit-only, no decisions made on output |
</threat_model>
