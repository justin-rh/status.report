<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AuditReport fields → Jinja2 template | Untrusted strings (hostname, cpu_model, os_version, app names) cross from Python into HTML rendering |
| Template file → renderer | Template loaded from package resources; could be tampered with if flash drive is writable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Injection (XSS) | `render_report` — all AuditReport string fields rendered into HTML | mitigate | `Environment(autoescape=True)` escapes `<`, `>`, `&`, `"`, `'` in ALL template variables automatically. Verified in RESEARCH.md Pitfall 2. A cpu_model like `Intel <i5>` becomes `Intel &lt;i5&gt;` in output. No manual escaping needed in template. |
| T-03-02-02 | Injection (Template) | `env.from_string(template_source)` — template source loaded from package resource | accept | Template is a static file shipped with the tool, not user-supplied content. In `--onedir` mode, template is a file in the bundle directory. Attacker would need write access to the bundle directory to tamper with it — equivalent to compromising the executable. Accepted risk for an internal IT tool. |
| T-03-02-03 | Spoofing | `disk_pct` — inline style `width: {{ disk_pct }}%` | accept | `disk_pct` is a Python float computed from `float / float` in `_build_context`. It is NOT user-supplied. Autoescape would also prevent `%` injection in a style value. Risk is negligible. |
| T-03-02-04 | Information Disclosure | `tests/test_renderer.py` — MOCK_REPORT contains realistic-looking machine data | accept | MOCK_REPORT uses fictional hostname `PHX-INV-003` and synthetic versions. No real machine data. Test file is development-only and not bundled into the exe. |
| T-03-02-05 | Denial of Service | `_load_template_source()` — template file missing at runtime | accept | If template file is absent (packaging error), `ir.files('renderer').joinpath().read_text()` raises `FileNotFoundError`. This propagates to the caller and produces a Python traceback — no data loss, no security impact. Phase 5 packaging must include the template (RESEARCH.md Pitfall 6). |
