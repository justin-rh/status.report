# Phase 3 — UI Design Contract

> Visual and interaction contract for the HTML Character Sheet Renderer.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> Stack: Pure HTML + embedded `<style>` block rendered via Jinja2. No component library,
> no npm, no external CSS files. Self-contained single-file HTML output is required for
> PyInstaller bundle compatibility. shadcn gate: NOT applicable.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — pure HTML/CSS in Jinja2 template |
| Preset | not applicable |
| Component library | none |
| Icon library | Unicode characters only (✓ U+2713, ✗ U+2717) |
| Font | system-ui, -apple-system, "Segoe UI", sans-serif (stack, no external load) |

**CSS delivery:** All styles embedded in a single `<style>` block inside the template.
No `<link>` to external stylesheets. No inline `style=""` attributes except for the
dynamic HP bar width (set via Jinja2 expression).

---

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/badge gap, tight inline padding |
| sm | 8px | Within-row cell padding, badge padding (horizontal) |
| md | 16px | Default section inner padding, table cell padding |
| lg | 24px | Between section blocks, card inner padding |
| xl | 32px | Major layout gap between character sheet panels |
| 2xl | 48px | Top/bottom page margin |
| 3xl | 64px | Not used in this phase |

Exceptions: HP bar height fixed at 20px (functional display constraint, not a spacing token).

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Stat values, table cell content, version strings |
| Label | 12px | 600 | 1.4 | Section header labels, field names (CPU, RAM, Disk), badge text |
| Heading | 20px | 600 | 1.2 | Section titles (STAT BLOCK, EQUIPMENT, QUEST STATUS) |
| Display | 28px | 600 | 1.1 | Character name (hostname) in page header |

Font stack: `system-ui, -apple-system, "Segoe UI", Arial, sans-serif`

**No serif fonts. No fantasy/decorative fonts.** The RPG aesthetic is carried entirely
by layout structure, color, and spacing — not typography.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a2e` | Page background, outer panel background |
| Secondary (30%) | `#16213e` | Section card backgrounds, table backgrounds |
| Surface accent | `#0f3460` | Section header background rows, panel borders |
| Body text | `#e0e0e0` | All body text, stat values, table content |
| Muted text | `#6b7280` | None-field em-dash `—`, supplementary labels |
| Heading text | `#a8b2d8` | Section title text rendered over secondary background |
| Border | `#2d3561` | Table borders, card borders, separator lines |

### Semantic / State Colors

| Role | Value | Reserved For |
|------|-------|-------------|
| Installed (green) | `#22c55e` | App badge background when `installed=True`; HP bar when disk >50% free |
| Warning (amber) | `#f59e0b` | HP bar when disk is 20–50% free |
| Missing / Low (red) | `#ef4444` | App badge background when `installed=False`; HP bar when disk ≤20% free |
| Quest Complete | `#22c55e` | Quest Status text + border when all apps installed |
| Quest Incomplete | `#ef4444` | Quest Status text + border when any apps missing |
| HP bar background | `#374151` | Empty/track portion of HP bar in all states |
| None bar (grey) | `#4b5563` | HP bar fill when `disk_total_gb` is None |

Accent (10%) reserved for: app installed badge, app missing badge, HP bar fill, Quest
Status text and border. These are the ONLY elements that use semantic color. Section
headers use `#0f3460` (surface accent), not semantic green/red/amber.

**60/30/10 split applied to surface area:**
- 60%: `#1a1a2e` page/outer background
- 30%: `#16213e` section cards and table bodies
- 10%: semantic state colors (badges, HP bar, quest footer) + `#0f3460` section header rows

## Component Specs

### Character Sheet Header

Layout: single-column centered block, `#16213e` card background, `lg` (24px) padding.

| Field | Label | Render Rule | Source |
|-------|-------|-------------|--------|
| Name | CHARACTER | `hostname` verbatim | AuditReport.hostname |
| Class | CLASS | `device_type` verbatim | ParsedHostname.device_type |
| Realm | REALM | `city` verbatim | ParsedHostname.city |
| Guild | GUILD | `department` (warehouse) or `company_code` (user laptop) or `—` | ParsedHostname |
| Station | STATION | Raw int or `—` | ParsedHostname.station |

Hostname renders in Display (28px/600) white. Field labels render in Label (12px/600)
`#a8b2d8`. Field values render in Body (14px/400) `#e0e0e0`.

### Stat Block

Layout: two-column CSS grid (label | value), `#16213e` background card, section header
row at `#0f3460`.

| Stat Row | Label | Value Format | None Fallback |
|----------|-------|-------------|---------------|
| CPU | CPU | CPU model string (e.g., "Intel Core i7-10700") | `—` in `#6b7280` |
| RAM | RAM | `X.X GB` (e.g., "16.0 GB") | `—` in `#6b7280` |
| Disk | DISK | `X GB total` (e.g., "476 GB total") | `—` in `#6b7280` |
| OS | OS | `{os_version} — Build {os_build}` combined | `—` in `#6b7280` |
| User | CURRENT USER | `current_user` string | `—` in `#6b7280` |

**HP Bar (Disk)** — displayed directly below the DISK row, spanning full width of value column:

- Track: 100% wide, 20px tall, background `#374151`, `border-radius: 4px`
- Fill: width set via Jinja2 inline style `style="width: {{ pct }}%"` — this is the only
  permitted inline style in the template
- Fill color: `#22c55e` when >50% free; `#f59e0b` when 20–50% free; `#ef4444` when ≤20% free
- Fill color when None: `#4b5563` (grey), width 100% (full grey bar)
- Label below bar: `X GB free / Y GB total` in Body (14px/400) `#e0e0e0`; or `—` when None

### Equipment List (App Slots)

Layout: table with columns: App Name | Status Badge | Version | Service State.
Section header row at `#0f3460`. Table body rows alternate between `#16213e` and `#1a1a2e`
(zebra stripe, 1-row offset).

| Column | Width | Content |
|--------|-------|---------|
| App Name | auto | App name string, Body (14px/400) `#e0e0e0` |
| Status | 80px | Badge: `✓ Installed` (green) or `✗ Missing` (red) |
| Version | 160px | Version string when installed; empty cell when missing |
| Service | 120px | Service state string (e.g., "Running", "Stopped") when present; empty cell otherwise |

**Badge spec:**
- Background: `#22c55e` (installed) or `#ef4444` (missing)
- Text: `#ffffff`, Label (12px/600), `border-radius: 4px`, padding `xs sm` (4px 8px)
- Icon: `✓` (U+2713) for installed, `✗` (U+2717) for missing
- Label text: "Installed" / "Missing"

### Quest Status Footer

Layout: full-width banner block, `border-radius: 8px`, `md` (16px) padding, centered text.

| State | Background | Border | Text | Text Color |
|-------|-----------|--------|------|------------|
| Complete | `#14532d` | 2px solid `#22c55e` | `QUEST COMPLETE` | `#22c55e` |
| Incomplete | `#450a0a` | 2px solid `#ef4444` | `QUEST INCOMPLETE — X app(s) missing` | `#ef4444` |

Text renders in Heading (20px/600). "X" is the integer count of apps where `installed=False`.

**Source (CONTEXT.md D-11).**

### Chronicle (Footer)

Single line below Quest Status: `Generated: {timestamp}` in Body (14px/400) `#6b7280`,
centered. Timestamp is `AuditReport.timestamp` formatted as ISO 8601 local time
(e.g., `2026-05-04 22:10:00`).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Page title (`<title>`) | `Status Report — {hostname}` |
| Header section label | `CHARACTER SHEET` |
| Stat block section label | `STAT BLOCK` |
| Equipment section label | `EQUIPMENT` |
| Quest status section label | `QUEST STATUS` |
| Quest complete | `QUEST COMPLETE` |
| Quest incomplete | `QUEST INCOMPLETE — {n} app(s) missing` |
| None field display | `—` (em-dash, U+2014) |
| HP bar None label | `—` |
| HP bar healthy label | `{free} GB free / {total} GB total` |
| Badge installed | `✓ Installed` |
| Badge missing | `✗ Missing` |
| Service state label | `{state}` verbatim (e.g., "Running", "Stopped") |
| Chronicle label | `Generated: {timestamp}` |
| OS combined | `{os_version} — Build {os_build}` |

**No fantasy vocabulary in any user-visible string.** Labels use real IT terminology
exactly as specified in CONTEXT.md decisions D-01 through D-06.

Empty states: This phase has no empty state per se — every field always renders,
using `—` for None values (D-12). The only absence state is "no apps" in the equipment
table, which would show zero rows. This state is not exercised in Phase 3 (mock data
uses all 11 apps, D-10).

Error states: No user-facing error strings in the rendered HTML. Rendering errors
propagate as Python exceptions before the file is written. The HTML file either
exists and is valid, or it does not exist.

Destructive actions: None. This phase produces a read-only output file. No delete,
reset, or confirmation interactions exist.

## Jinja2 Template Conventions

| Pattern | Rule |
|---------|------|
| None to em-dash | Jinja2 filter `{{ value | default('—') }}` applied to all nullable fields |
| HP bar width | Inline style only: `style="width: {{ disk_pct }}%"` on the fill `<div>` |
| HP bar color class | CSS class applied: `hp-green` / `hp-amber` / `hp-red` / `hp-none` — resolved in template logic |
| Badge class | CSS class applied: `badge-installed` / `badge-missing` — resolved in template loop |
| Quest class | CSS class applied: `quest-complete` / `quest-incomplete` — resolved in template logic |
| Zebra stripe | CSS `:nth-child(even)` on `<tr>` elements in equipment table |
| No external resources | No `<link>`, no `<script src>`, no `@import`, no CDN URLs anywhere in template |

## Layout Structure (HTML Skeleton)

```
<body>                          <!-- bg: #1a1a2e -->
  <div class="sheet">           <!-- max-width: 900px, centered, padding: 2xl (48px) -->
    <div class="header">        <!-- CHARACTER SHEET header card -->
    <div class="stat-block">    <!-- STAT BLOCK section card -->
    <div class="equipment">     <!-- EQUIPMENT section card -->
    <div class="quest-status">  <!-- QUEST STATUS banner -->
    <div class="chronicle">     <!-- Generated: timestamp -->
  </div>
</body>
```

Page max-width: 900px. Centered with `margin: 0 auto`. No responsive breakpoints —
this is a print-to-USB tool viewed in a browser on a desktop machine. Single-column
layout throughout.
