def _collect_cpu_model(report: AuditReport) -> None:
    try:
        import wmi
        c = wmi.WMI()
        processors = c.Win32_Processor()
        if processors:
            report.cpu_model = processors[0].Name.strip()
    except Exception as exc:
        report.collection_errors.append(f"CPU model collection failed (WMI): {exc}")
```
The entire wmi import + instantiation + query is wrapped in one try/except. If any step fails, cpu_model stays None (model default) and one error is appended (D-02).

Implement _collect_memory_and_disk(report) with psutil (per D-05):

def _collect_memory_and_disk(report: AuditReport) -> None:
    import psutil
    # RAM — always works at standard user privilege
    report.ram_gb = round(psutil.virtual_memory().total / (1024 ** 3), 1)
    # Disk — target C:\ primary volume
    try:
        disk = psutil.disk_usage('C:\\')
        report.disk_total_gb = round(disk.total / (1024 ** 3), 1)
        report.disk_free_gb = round(disk.free / (1024 ** 3), 1)
    except (FileNotFoundError, PermissionError, Exception) as exc:
        report.collection_errors.append(f"Disk usage collection failed: {exc}")
```
ram_gb is NOT wrapped in try/except (psutil.virtual_memory() never fails on standard user). Disk is wrapped because unusual configurations (no C:\, junction issues) can cause FileNotFoundError.

```python
def collect_profiles(report: AuditReport) -> None:
    """Populates local_profiles from HKLM ProfileList registry (D-08, D-03, D-04).
    Filters system SIDs. Extracts username from last path segment.
    """
    try:
        report.local_profiles = _enumerate_profiles()
    except Exception as exc:
        report.collection_errors.append(f"Profile enumeration failed: {exc}")


def _enumerate_profiles() -> list[str]:
    """Read HKLM ProfileList, skip system SIDs, return list of usernames."""
    profiles: list[str] = []
    try:
        with winreg.OpenKey(winreg.HKEY_LOCAL_MACHINE, PROFILE_LIST_KEY) as key:
            i = 0
            while True:
                try:
                    sid = winreg.EnumKey(key, i)
                    i += 1
                    if sid in SYSTEM_SIDS:
                        continue
                    try:
                        with winreg.OpenKey(key, sid) as sid_key:
                            path, _ = winreg.QueryValueEx(sid_key, 'ProfileImagePath')
                            expanded = winreg.ExpandEnvironmentStrings(path)
                            username = expanded.rstrip('\\').split('\\')[-1]
                            if username:
                                profiles.append(username)
                    except (FileNotFoundError, OSError):
                        continue  # Skip unreadable SID subkey silently
                except OSError:
                    break  # EnumKey raises OSError when index exhausted — normal loop end
    except OSError as exc:
        raise  # Re-raise so collect_profiles catches and logs to collection_errors
    return profiles
```

CRITICAL: winreg is stdlib — it is available on Windows Python 3.12 without installation. Import it at module top level (not lazy), since it is always available on the target platform.

The `winreg` import is at the top of the module (not inside the function) because it is stdlib and always present on Windows. Only `wmi` and `psutil` use lazy imports inside their functions.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Registry → application | ProfileImagePath values read from HKLM are attacker-controllable if the machine is compromised; we consume the path string for display only |
| WMI COM → application | Win32_Processor.Name is read-only hardware data; no user input involved |
| Environment variable → application | os.environ['USERNAME'] comes from Windows session; no user-controlled external input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | ProfileImagePath registry value | accept | Value is used only as a display string for IT staff; not executed, not written anywhere, not used in path operations that could escape. Last-segment extraction via `split('\\')[-1]` is safe even with path traversal attempts — the result is a simple username string. |
| T-02-02 | Denial of Service | wmi.WMI() COM call | mitigate | Entire WMI block wrapped in `try/except Exception`; a hung or unavailable WMI service degrades gracefully to cpu_model=None + error logged. No timeout needed — WMI client calls fail fast when the service is down. |
| T-02-03 | Information Disclosure | collection_errors list | accept | collection_errors is written to the AuditReport for IT staff consumption. Exception messages from wmi/psutil/winreg may include system path fragments, which is acceptable for an internal IT audit tool with no network transmission. |
| T-02-04 | Elevation of Privilege | winreg.OpenKey(HKEY_LOCAL_MACHINE, ...) | accept | Read-only access to HKLM ProfileList. Standard users have read access to this key on all Windows 10/11 machines. No write operations performed. No registry modification possible from this code path. |
| T-02-05 | Spoofing | os.environ['USERNAME'] | accept | USERNAME reflects the current Windows session identity, set by the OS. In an IT audit context this is expected behavior — IT staff see who ran the tool. No authentication decision is made from this value. |
</threat_model>
