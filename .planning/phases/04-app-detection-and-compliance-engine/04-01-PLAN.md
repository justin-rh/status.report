       Iterates UNINSTALL_PATHS. For each path, opens root key, loops EnumKey (OSError=exhausted), opens each subkey, reads DisplayName via QueryValueEx. If any keyword (case-insensitive substring) matches, reads DisplayVersion (catching FileNotFoundError/OSError), returns (True, version). Catches (FileNotFoundError, OSError) at root open and at subkey open level, continues to next. Returns (False, None) if all paths exhausted.

    4. _read_service_start(service_name: str) -> str | None:
       Opens HKLM\SYSTEM\CurrentControlSet\Services\{service_name}, reads Start DWORD, returns _START_MAP.get(int(val)). Catches (FileNotFoundError, OSError, ValueError, TypeError), returns None on any failure.

    5. _detect_msix(family_prefix: str) -> tuple[bool, str | None]:
       Opens HKCU _MSIX_REPO_PATH, loops EnumKey, checks startswith(family_prefix). On match, splits key name on "_", returns (True, parts[1] if len(parts) >= 2 else None). Catches (FileNotFoundError, OSError), returns (False, None).

    6. _detect_one_app(spec: dict, report: AuditReport) -> None:
       Logic flow (per D-02, D-03, D-07, D-08, D-11):
       - Initialize: installed=False, version=None, service_state=None, detection_method='registry'
       - If "msix_family_prefix" in spec: call _detect_msix(spec["msix_family_prefix"]); if installed, set installed/version; detection_method stays 'registry' (MSIX is still a registry read)

       - report.apps.append(AppStatus(name=spec["name"], installed=installed, version=version, service_state=service_state, detection_method=detection_method))

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Registry → Python | winreg returns strings/DWORDs from HKLM and HKCU — values are read from the local machine registry, not user input |
| Filesystem → Python | Path.exists() on a hardcoded path — no user-supplied path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | _search_uninstall_keys: DisplayName value read from registry | accept | Registry values are read as strings and compared via substring match — never executed, never deserialized. An attacker who can write to HKLM Uninstall keys already has admin rights and there is no further privilege to gain from a false keyword match. |
| T-04-01-02 | Information Disclosure | collect_apps exception handling: exc appended to report.collection_errors | accept | Exception message strings are appended to an in-memory list never sent off-machine. No PII. Error detail stays on the flash drive output only. |
| T-04-01-03 | Tampering | _detect_msix: HKCU AppModel Repository key names read from registry | accept | Key names are split on "_" and the second segment is returned as version string — never executed. An attacker controlling HKCU can already modify the logged-in user's environment. |
| T-04-01-04 | Tampering | MERP filesystem_path: hardcoded path checked via Path.exists() | accept | Path is a hardcoded string constant in APP_SPECS, not user-supplied. Path.exists() does not read file contents or execute. No traversal risk. |
| T-04-01-05 | Denial of Service | _read_service_start: repeated OpenKey on CSFalconService | accept | Single synchronous read per execution, not in a loop. No retry logic. Cannot be triggered repeatedly from outside. |
</threat_model>
